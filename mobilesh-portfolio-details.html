Project Overview

This project involves API testing for the "Second Hand" e-commerce mobile app. The goal was to ensure that all API endpoints functioned properly under various conditions, including security, performance, and functional testing. The testing was carried out using Postman, and results were compiled into a detailed report.

Configuration:

Collection Creation: A Postman Collection was created based on the API documentation.

API Endpoints Covered:

  • Auth:
    • POST /auth/register
    • POST /auth/login
    • GET /auth/user
    • PUT /auth/user
    • PUT /auth/change-password
  • Seller - Product:
    • POST /seller/product
    • GET /seller/product
    • GET /seller/product/{id}
    • PUT /seller/product/{id}
    • PATCH /seller/product/{id}
    • DELETE /seller/product/{id}
  • Seller - Order:
    • GET /seller/order
    • GET /seller/order/{id}
    • PATCH /seller/order/{id}
    • GET /seller/order/product/{product_id}
  • Seller - Category:
    • GET /seller/category
    • GET /seller/category/{id}
  • Seller - Banner:
    • GET /seller/banner
    • GET /seller/banner/{id}
  • Buyer - Product:
    • GET /buyer/product
    • GET /buyer/product/{id}
  • Buyer - Order:
    • POST /buyer/order

Testing and Results:

Assertion Testing: A total of 43 test cases were created, covering both positive and negative scenarios. The results for all tests were Pass, indicating the API is functioning as expected.

Web Service Testing: Apart from functional testing, I conducted web service tests focusing on:

  • Security: Ensuring the API is secure and free from vulnerabilities.
  • Request/Response Validation: Verifying that the data formats are correct and consistent with the API documentation.
  • Performance Testing: Evaluating server performance, response times, and stability under different loads.

Conclusion:

All tests have been successfully completed with positive results. The API for the Second Hand mobile app is fully functional, secure, and ready for release. The performance tests also show the server can handle a large number of requests without issues. For detailed results, refer to the 3rd sheet of the Newman test results.
