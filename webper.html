Test Overview

This performance test is focused on the "Second Hand" e-commerce website, with 14 API requests tested under a simulated load to evaluate the stability and performance under stress conditions.

Test Configuration

  • Number of Threads (Users): 20
  • Ramp-Up Period: 1 second
  • Loop Count: 10 iterations

Testing Process:

The test scenario includes loading 14 API requests based on the backend documentation, such as authentication, product categories, offers, products, and user profiles.

Results:

  • All requests completed successfully under a load of 20 concurrent users.
  • The response times for critical endpoints (like login and product retrieval) remained stable, averaging 2-3 seconds.
  • No server errors (5xx) or client-side errors (4xx) were encountered during the test.
  • All requests passed the performance thresholds defined in the project requirements.

Conclusion:

The performance testing for the Second Hand website was successful, with the website proving capable of handling the expected user load with consistent response times. No major performance bottlenecks or critical issues were identified during the testing phase.
